什么是Dockerfile

Dockerfile是由一系列命令和参数构成的脚本，这些命令应用于基础镜像并最终创建一个
新的镜像。

  • 对于开发人员：可以为开发团队提供一个完全一致的开发环境；
  • 对于测试人员：可以直接拿开发时所构建的镜像或者通过Dockerfile文件构建一个新
    的镜像开始工作了；
  • 对于运维人员：在部署时，可以实现应用的无缝移植。
命令 作用
FROM image_name:tag 定义了使用哪个基础镜像启动构建
MAINTAINER user_name 声明镜像的创建者
ENV key value 设置环境变量，（可以写多条）
RUN command 构建镜像时运行的Shell命令，是Dockerfile的核心部分（可以写多条）
ADD source_dir/filedest_dir/file 拷贝文件或目录到镜像中，如果是一个压缩文件， 将会在复制后自动解压
COPY source_dir/filedest_dir/file 和ADD相似，只是不支持自动下载和解压
WORKDIR path_dir 设置工作目录，类似cd
EXPOST port1 port2 用来指定端口，使容器内的应用通过端口和外界交互
CMD argument 启动容器时执行的Shell命令，构建容器时使用会被docker run参数覆盖
ENTRYPOINT argument 和CMD相似，只是由ENTRYPOINT启动的程序不会被docker run命令行指定的参数所覆盖
VOLUME 指定容器挂载点到宿主机自动生成的目录或其他容器，一般不会在Dockerfile用到，更常见的还是在docker run的时候指定-v数据卷。

例子

使用DockerFile来创建JDK1.8镜像

  1. 创建目录
    mkdir –p /home/temp/dockerjdk8
  2. 下载jdk-8u171-linux-x64.tar.gz并上传到服务器（虚拟机）中的/home/temp/dockerjdk8目录
    scp jdk-8u171-linux-x64.tar.gz root@ailyun1
  3. 创建文件Dockerfile 
     cd /home/temp/dockerjdk8 <br/>
 vim Dockerfile
 #依赖镜像名称和ID
 FROM centos:7
 #指定镜像创建者信息
 MAINTAINER ITCAST
 #切换工作目录 相当于cd
 WORKDIR /usr
 RUN mkdir /usr/local/java
 #ADD 是相对路径jar,把java添加到容器中
 ADD jdk‐8u171‐linux‐x64.tar.gz /usr/local/java/
 #配置java环境变量
 ENV JAVA_HOME /usr/local/java/jdk1.8.0_171
 ENV JRE_HOME $JAVA_HOME/jre
 ENV CLASSPATH
 $JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib:$CLASSPATH
 ENV PATH $JAVA_HOME/bin:$PATH
  4. 执行命令构建镜像
    docker build ‐t='jdk1.8'
